# Milestone 18 — Contextual navigation and Capacity workload chart

## Navigation audit

Person-Month Allocation now uses the existing `context` query mechanism with a strict
whitelist. `project-overview` and `capacity` are the only contextual origins; the
legacy `project` value is normalized for backward compatibility. The only preserved
state is the validated canonical year. URLs are generated by `UrlGenerator`, so the
configured application base path is retained. Arbitrary URLs, referrers, browser
history and unsupported query parameters are not used.

| Origin | Allocation entry point | Return destination |
| --- | --- | --- |
| Project Overview | Participant name | Same project and year |
| Project Overview | Monthly value or divergent value | Same project and year |
| Capacity | Supported context (no direct allocation entry currently exists) | Same Capacity year |
| People list | No allocation entry currently exists | Not applicable |
| Person profile | No Person allocation profile currently exists | Not applicable |
| My Profile | No allocation link currently exists | Not applicable |
| Direct or invalid URL | Allocation detail/list | Participant's canonical allocation list |

The context is propagated through the allocation list, detail, create/edit forms,
validation rendering, successful redirects, cancellation and removal confirmation.
The contextual header and breadcrumb identify the Person, project and month.

The focused review of Configure Project, Details, Work Packages and Participants
found their Milestone 14 year-aware navigation intact. No parallel return mechanism
was introduced.

## Capacity workload chart

Capacity now begins with an accessible annual workload chart, sorted from lowest
utilization. Each row shows the Person, a bordeaux allocated/capacity bar, utilization
percentage and exact available hours. Over-capacity values use the warning color.
The bars are native semantic HTML/CSS with screen-reader labels and a responsive
mobile layout. The detailed monthly table now also includes annual Allocated and
Remaining totals.

The chart is a direct presentation of existing monthly allocations and effective
capacity. It adds no effort model, threshold or mutation rule.
